## 2. Your Project (Phase IV)

Your assignment implementation will be divided into two main parts: Front-End (GUI) and Back-End development. It will be assessed during the last lab.

Below are the guidelines and expectations for each part:

---

## 2.1 Back-End Implementation

- Finalize the design and implement the core logic, data processing, and server-side functionality of the system. *(Using a database is not essential.)*

- **Customization Requirement:**  
  Clearly define any changes or extensions you applied to existing open-source tools. This may include modifications to:
    - Algorithms
    - Data schemas
    - API routes
    - Security protocols
    - Other back-end elements

  Your work should demonstrate a clear understanding of the systems used. During the Sprint 4 lab, you will be asked to explain and justify these customizations.

- **Understanding of Modifications:**  
  You should be able to clearly explain the changes made to any open-source components used, including how and why they were adapted. These explanations will be assessed during the Sprint 4 demo.

- **Back-End Integration:**  
  You should be able to explain how your back-end integrates with the front-end and handles data requests and responses. You will be asked questions about this during the Sprint 4 lab, and your answers will be evaluated.

---

## 2.2 Front-End (GUI) Implementation

- Develop the graphical user interface that users will interact with to access all the system’s functionality.

- You are allowed to use open-source libraries or frameworks for GUI development, but the system must be tailored to suit the needs of the assignment.
